Plan de Validación

Verifica consistencia, distribución e integridad referencial de ambos esquemas de sharding

GET /horizontal/validate · GET /vertical/validate Endpoints de validación automatizada

9.1 · Validación Sharding Horizontal

Distribución uniforme Verificar ~33% por shard

Insertar múltiples registros y comprobar que la función hash distribuye homogéneamente entre los 3 shards.

Consistencia del hash Mismo ID → mismo shard siempre

El mismo ID debe caer siempre en su shard canónico. No pueden existir duplicados entre shards.

ID 5 → 5 % 3 = 2 ID 5 siempre → Shard 2

9.2 · Validación Sharding Vertical

Integridad referencial Ambas tablas sincronizadas

Verificar que todo usuario en usuarios_perfil tiene su contraparte en usuarios_actividad y viceversa.

Reconstrucción con JOIN Ninguna columna se pierde

El JOIN reconstituye la fila completa. La información no se pierde al dividir verticalmente.

usuarios_perfil (5 cols) + usuarios_actividad (5 cols) → 9 cols totales
